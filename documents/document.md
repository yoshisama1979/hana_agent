# 法人決算支援ツール 仕様書

## 概要

月次の仕訳作業を自動化するCLIツール。
複数のデビットカード明細（CSV）と freee 仕訳帳（CSV）を照合し、
勘定科目ルールに基づいて仕訳済み結果を出力する。

現在対応しているカード：

- **りそなデビットカード**（口座: りそな銀行大手支店普通0073514）
- **住信SBIネット銀行デビットカード**（口座: 住信SBI106代表口座 - 円普通2212165）

### 全体方針：複数視点で仕訳帳を消化する

仕訳帳の全行を、複数の照合タスクで段階的に消化していく。
各照合タスクでマッチした取引は当該タスクの出力CSVに振り分けられ、
最終的に残った未照合行が `仕訳帳のみ.csv` に集約される。
理想形は `仕訳帳のみ.csv = 0件`。

```
仕訳帳全件
  ├─ カード照合（りそな） → output/risona/カード_照合済み で消化
  ├─ カード照合（SBI）　   → output/sbi/カード_照合済み で消化
  ├─ 売掛金照合           → output/receivable/売掛金_照合済み で消化
  ├─ 振込照合（将来）　    → 振込_照合済み で消化
  ├─ 給与照合（将来）　    → 給与_照合済み で消化
  └─ 仕訳帳のみ             ← 0 を目指す
```

---

## 機能一覧

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| F-01 | VISAデビ抽出 | 仕訳帳から「★要確認 × VISAデビ始まり」の行を抽出 |
| F-02 | カード照合 | カード明細 × 仕訳帳を金額＋日付で照合（複数カード対応） |
| F-03 | 勘定科目推定 | ルールファイルの正規表現で利用内容から勘定科目を推定 |
| F-04 | 結果出力 | 照合結果・未入力・仕訳帳のみをCSVに分けて出力 |
| F-05 | 売掛金照合 | 仕訳帳内の 請求（借方=売掛金）⇄ 入金（貸方=売掛金）を補助科目＋金額で消し込む |

## F-02 カード照合処理 詳細仕様

### カードプロファイル

各カードは「カードプロファイル」として定義し、共通ロジックでまとめて処理する。

| 項目 | りそな | SBI |
|---|---|---|
| カードID（出力ディレクトリ名） | `risona` | `sbi` |
| 仕訳帳の補助科目名 | `りそな銀行大手支店普通0073514` | `住信SBI106代表口座 - 円普通2212165` |
| 明細CSVパス | `data/accounting/risona_debit.csv`（単一ファイル） | `data/accounting/meisai_*.csv`（複数ファイル glob） |
| 明細CSVエンコード | CP932 | CP932 |
| 利用日カラム | `利用日` | `お取引日` |
| 利用内容カラム | `利用内容` | `お取引内容` |
| 金額カラム | `金額` | `お取引金額` |
| ステータス列 | あり（`ステータス`） | なし（全行を「確定」扱い） |
| 日付許容ズレ（`date_tolerance_days`） | `0`（厳格一致のみ） | `14`（最大±14日） |
| 手数料カラム（`fee_cols`） | なし（空タプル） | `("海外事務手数料",)` |

### 入力ファイルの読み込み

- **りそな**：単一CSVを読み込む
- **SBI**：`meisai_*.csv` をすべて glob で読み込み、結合する。月またぎで重複する行（取引日 × 利用内容 × 金額が一致）は除外する

### 仕訳帳の対象行（カード照合スコープ）

各カードについて、以下の条件を満たす仕訳帳行を照合対象とする。

- `借方補助科目` または `貸方補助科目` がそのカードの口座名
- かつ反対側の `勘定科目` が `"普通預金"` または `"現金"` ではない（口座間振替・現金引出を除外）

### 照合キー

- **金額**：明細の金額の絶対値 == 仕訳帳 `借方金額(円)` の絶対値
  - 明細側は返金がマイナス値、仕訳帳側は借方/貸方で方向を表現するため絶対値で比較
- **日付**：明細の利用日 == 仕訳帳 `取引日`（フォーマット正規化後に比較）
  - SBIデビットカードは「カード利用日」と「口座引落日」が最大数日ズレるため、
    カードプロファイルで `date_tolerance_days` を指定して許容ズレを設定する。
    詳細は次節「段階マッチング」を参照。

### 段階マッチング（日付許容ズレ対応）

`date_tolerance_days = N` を指定したカードでは、ズレの少ないものから順に消化する
段階マッチングを行う。各段階で消化できなかった行が次段階に持ち越される。

```
段階0: 日付ズレ 0日（厳格一致）
   ↓ 残った行
段階1: 日付ズレ ±1日（同金額）
   ↓ 残った行
段階2: 日付ズレ ±2日（同金額）
   ↓ ...
段階N: 日付ズレ ±N日（同金額）
   ↓ 残った行
最終的に「カード_未入力 / 重複要確認 / 仕訳帳のみ」へ振り分け
```

各段階のマッチング規則：

- **段階0（厳格一致）**：同日・同金額でグループ化し、件数一致なら 1:1 または N:N（要目視）でマッチ。件数不一致は「重複要確認」へ。
- **段階 1 以上（日付ズレ）**：金額が一致し、日付差の絶対値がちょうど段階値となる行同士を greedy に 1:1 でペアリング。複数候補がある場合は先頭から順に消費する。

`date_tolerance_days = 0` のカード（りそな）は段階0のみ実行され、現行と挙動は変わらない。

### 手数料込み金額でのフォールバック照合

SBIの海外通貨取引（VERCEL / OPENAI / CLAUDE.AI 等）では、
明細上の「お取引金額」と仕訳帳の金額が一致しないことがある。
仕訳帳側で「お取引金額 + 海外事務手数料」の合計額で1行に計上されているためである。

このケースを救うため、カードプロファイルで `fee_cols` を指定すると、
段階マッチングの各段階で「本体金額」と「本体+手数料合算」の2戦略を順に試す。

**「日付の近さ」を最優先**するため、段階（日付ズレ日数）を外側ループ、
金額戦略（本体／本体+手数料）を内側ループとする：

```
段階0:
   本体 でマッチ
   ↓ 残り
   本体+手数料 でマッチ（fee_cols 指定時かつ手数料>0の行のみ）
   ↓ 残り
段階1:
   本体 でマッチ
   ↓ 残り
   本体+手数料 でマッチ
   ↓ 残り
...
段階N:
   本体 → 本体+手数料
   ↓ 残り
最終的に「カード_未入力 / 重複要確認」へ
```

この入れ子順により、たとえば「同日同額（本体+手数料）」と「12日違いの同金額（本体）」
の両方が候補になる場合、前者（段階0/本体+手数料）が後者（段階12/本体）より先に消化されるため、
**日付の近いペアが正しく優先される**。

`fee_cols` を持たないカード（りそな）は内側ループの 本体+手数料 段階をスキップする。
出力の `金額種別` 列で本体マッチ（`本体`）と手数料込みマッチ（`本体+手数料`）を区別する。

| 元フォーマット | 正規化後 |
|----------------|----------|
| `2025年04月07日`（りそな） | `2025-04-07` |
| `2025/04/07`（SBI／仕訳帳） | `2025-04-07` |

### りそな明細のステータスフィルタ

| ステータス | 扱い |
|---|---|
| 確定 | 照合対象 |
| 未確定（承認分）等 | `カード_未確定.csv` に退避（仕訳化前なので照合対象外） |
| 決済不可 | `カード_対象外.csv` に退避（取引不成立のため対象外） |

SBIにはステータス列がないため、全行を「確定」扱いとする（`カード_未確定.csv` `カード_対象外.csv` は出力しない）。

### 出力ファイル一覧

各カードの照合結果は `data/accounting/output/<カードID>/` 配下に出力する。
`仕訳帳のみ.csv` のみ全カード共通で `data/accounting/output/` 直下に出力する。

| 出力ファイル | 出力先 | 内容 |
|------|------|------|
| `カード_照合済み.csv` | `output/<カードID>/` | 金額＋日付で一致した明細と仕訳帳の対 |
| `カード_照合済み_コンパクト.csv` | `output/<カードID>/` | 照合済みを入力支援用カラムに絞り込み |
| `カード_重複要確認.csv` | `output/<カードID>/` | 同日・同金額が複数件で1対1にならず要目視判定 |
| `カード_未入力.csv` | `output/<カードID>/` | 明細（確定）にあって仕訳帳に未入力 |
| `カード_未確定.csv` | `output/<カードID>/` | 明細でステータスが「確定」以外（りそなのみ） |
| `カード_対象外.csv` | `output/<カードID>/` | 明細でステータスが「決済不可」（りそなのみ） |
| `仕訳帳のみ.csv` | `output/` 直下 | 全仕訳帳から全カードの照合済み・重複要確認の取引Noを除いた残り |

### 仕訳帳のみ.csv の定義

```
仕訳帳のみ.csv = 全仕訳帳 − 全カードの (カード_照合済み の取引No + カード_重複要確認 の取引No)
```

カード照合（全カード分）で消化されなかった全仕訳帳行が集約される。本ツールの
スコープ外（社会保険料引落、振込手数料、法人税納付など）も含まれ、将来の振込照合・
給与照合などのツールで段階的に消化していく対象となる。

### 最終評価

カード照合の完了判定は、各カードごとに `カード_重複要確認` と `カード_未入力` の合計で行う。
両方が0件なら `〇`。`仕訳帳のみ.csv` の件数は別途表示するが、評価対象には
含めない（他の照合ツールで消化すべき行のため）。

`date_tolerance_days > 0` のカードでは、段階別の消化件数も併記する。

```
カード照合(risona): 〇 すべて照合済み
  本体 完全一致: 426件
カード照合(sbi):    × 未解決 22件
  本体 完全一致: 293件
  本体 日付ズレ ±1日: 22件
  本体 日付ズレ ±2日: 5件
  ...
  本体+手数料 完全一致: 12件
  本体+手数料 日付ズレ ±1日: 18件
  カード_重複要確認: 6件
  カード_未入力: 16件
仕訳帳のみ: 1080件（カード照合外・他ツールでの照合待ち）
```

## F-03 勘定科目推定 詳細仕様

### ルールファイル

- パス：`accounting/rules/merchant_rules.yml`
- フォーマット：YAML
- 全カード共通で使用する（カードを問わず利用内容パターンが共通のため）

```yaml
rules:
  - match: "ZOOM"          # 正規表現（大文字小文字・全角半角を正規化後にマッチ）
    勘定科目: 消耗品費
    補助科目: Zoom
    税区分: 課税仕入
```

### マッチング手順

1. 明細の利用内容を **全角→半角** に正規化
2. ルールを上から順に評価し、**最初にマッチしたルールを採用**
3. どのルールにもマッチしない場合は `勘定科目 = "★ルール未登録"` とする

仕訳帳側で freee が VISA デビ取込時に勘定科目未確定として入れる `★要確認` と
区別するため、classifier 由来のデフォルト値は `★ルール未登録` を使う。

### 全角→半角正規化

- `unicodedata.normalize("NFKC", text)` を使用

## F-04 結果出力 詳細仕様

### カード_照合済み.csv

明細の全カラム＋仕訳帳全カラム（結合時に重複しないため `_journal` サフィックスは付かない）
＋勘定科目推定結果（`推定入力名` `推定勘定科目` `推定補助科目` `推定税区分`）
＋入力補助列（`利用内容（入力）` `借方勘定科目（入力）` `補助科目（入力）`）
＋ `要目視確認` フラグ（同日同額の複数件マッチ時に「要確認」）
＋ `照合方法` 列（段階0なら `完全一致`、段階1以上なら `日付ズレ`）
＋ `日付ズレ日数` 列（段階値そのもの。0, 1, ..., N）
＋ `金額種別` 列（パス1は `本体`、パス2は `本体+手数料`）

### カード_照合済み_コンパクト.csv

freeeへの入力作業時に確認しやすいカラムのみ抽出した版。
カラム順は `accounting/reconcile.py` の `COMPACT_COLS` で定義。
カードによってカラム名が異なるため、カードプロファイル側でカラム名のマッピングを定義する。

### カード_未入力.csv

明細の全カラム＋勘定科目推定結果。仕訳帳に未転記なので、
推定結果を参考に freee へ入力していく。

### カード_未確定.csv / カード_対象外.csv

明細の全カラム（フィルタのみ、加工なし）。
ステータス列を持つカード（りそな）でのみ出力される。

### 仕訳帳のみ.csv

仕訳帳の全カラム（フィルタのみ、加工なし）。

## F-05 売掛金照合 詳細仕様

仕訳帳内に閉じた突き合わせで、売掛金の「請求」と「入金」をペアリングして消し込む。
カード照合とは独立した処理で、明細CSVは不要。

### 対象行

仕訳帳のうち、以下のいずれかに該当する行：

- **請求側**：`借方勘定科目 = "売掛金"`
- **入金側**：`貸方勘定科目 = "売掛金"`

### 照合キー

- **補助科目**：請求側の `借方補助科目` == 入金側の `貸方補助科目`（取引先）
- **金額**：請求側の `借方金額(円)` == 入金側の `貸方金額(円)`（絶対値で比較）
- **補助科目が空欄の行**は対象外（取引先が特定できないため誤マッチを避ける）

### マッチング手順

```
Step 1: (補助科目, 金額) でグループ化
Step 2: 各グループ内で：
   2-1: 請求側を 取引日 昇順にソート
   2-2: 入金側を 取引日 昇順にソート
   2-3: 請求から順に処理し、その日付より「後」で未消費の入金を最も近い順に1件選んでペアリング
        （請求日 < 入金日 が必須。請求日 ≧ 入金日 のペアは作らない）
   2-4: 上記でペアにできなかった請求・入金は未消し込みのまま残す
```

### 出力ファイル一覧

| 出力ファイル | 出力先 | 内容 |
|------|------|------|
| `売掛金_照合済み.csv` | `output/receivable/` | 請求行と入金行のペア |

未消し込みの売掛金行（前年度分入金・次年度繰越請求など）は `仕訳帳のみ.csv` に
そのまま残す。本仕様の初版では「確実に消せるもの」だけを処理対象とする。

### 消化分の取扱い

`仕訳帳のみ.csv` の集計時、カード照合の消化分に加えて
売掛金照合でペアになった請求／入金行の取引Noも除外する。

```
仕訳帳のみ.csv = 全仕訳帳 − (全カード照合の消化分 + 売掛金照合の消化分)
```

## CLIインターフェース

すべてのオプションは省略可能。省略時は下記デフォルトが適用され、全カードを順次照合する。

```bash
# F-01: VISAデビ抽出
python -m accounting.extract_visa_debit [入力CSV] [出力CSV]

# F-02〜04: 全カードまとめて照合＋勘定科目推定
python -m accounting.reconcile \
  [--journal data/accounting/仕訳帳.csv] \
  [--rules   accounting/rules/merchant_rules.yml] \
  [--output  data/accounting/output/]
```

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `--journal` | `data/accounting/仕訳帳.csv` | freee仕訳帳CSV（CP932） |
| `--rules` | `accounting/rules/merchant_rules.yml` | 勘定科目ルールYAML |
| `--output` | `data/accounting/output/` | 結果CSVの出力先ディレクトリ |

明細CSVのパスはカードプロファイルで定義されるため、CLIオプションには含まない。
通常運用では引数なしで実行可能。

```bash
python -m accounting.reconcile
```

## 異常系・バリデーション

| 条件 | 挙動 |
|------|------|
| 入力ファイルが存在しない | `FileNotFoundError` を raise し、メッセージを表示して終了 |
| SBI明細CSVが1件もマッチしない（glob空） | `FileNotFoundError` を raise し、メッセージを表示して終了 |
| 必須カラムが存在しない | `ValueError` を raise し、欠損カラム名を表示して終了 |
| ルールファイルが存在しない | 警告を表示し、全行を `★ルール未登録` として処理を継続 |
