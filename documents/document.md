# 法人決算支援ツール 仕様書

## 概要

月次の仕訳作業を自動化するCLIツール。
りそなデビットカード明細（CSV）と freee 仕訳帳（CSV）を照合し、
勘定科目ルールに基づいて仕訳済み結果を出力する。

---

## 機能一覧

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| F-01 | VISAデビ抽出 | 仕訳帳から「★要確認 × VISAデビ始まり」の行を抽出 |
| F-02 | 照合処理 | りそな明細 × 仕訳帳を金額＋日付で照合 |
| F-03 | 勘定科目推定 | ルールファイルの正規表現で利用内容から勘定科目を推定 |
| F-04 | 結果出力 | 照合結果・未照合・要確認をCSVに分けて出力 |

---

## F-02 照合処理 詳細仕様

### 入力ファイル

| ファイル | エンコード | 主要カラム |
|----------|------------|------------|
| りそなデビット明細CSV | CP932 | 利用日, 利用内容, 金額, 承認番号 |
| freee 仕訳帳CSV | CP932 | 取引日, 借方勘定科目, 摘要, 借方金額(円) |

### 仕訳帳の対象行

- `借方勘定科目 == "★要確認"`
- `摘要` が `"VISAデビ"` で始まる行

### 照合キー

- **金額**：りそな `金額` == 仕訳帳 `借方金額(円)`
- **日付**：りそな `利用日` == 仕訳帳 `取引日`（フォーマット正規化後に比較）

### 日付フォーマット正規化

| 元フォーマット | 正規化後 |
|----------------|----------|
| `2025年04月07日`（りそな） | `2025-04-07` |
| `2025/04/07`（仕訳帳） | `2025-04-07` |

### 照合結果の分類

| 分類 | 条件 | 出力ファイル |
|------|------|-------------|
| 照合済み | 金額＋日付で1対1一致 | `output/matched.csv` |
| 重複照合 | 同日・同金額が複数件 | `output/duplicates.csv` |
| りそなのみ | りそなにあって仕訳帳にない | `output/risona_only.csv` |
| 仕訳帳のみ | 仕訳帳にあってりそなにない | `output/journal_only.csv` |

## F-03 勘定科目推定 詳細仕様

### ルールファイル

- パス：`accounting/rules/merchant_rules.yml`
- フォーマット：YAML

```yaml
rules:
  - match: "ZOOM"          # 正規表現（大文字小文字・全角半角を正規化後にマッチ）
    勘定科目: 消耗品費
    補助科目: Zoom
    税区分: 課税仕入
```

### マッチング手順

1. りそな `利用内容` を **全角→半角** に正規化
2. ルールを上から順に評価し、**最初にマッチしたルールを採用**
3. どのルールにもマッチしない場合は `勘定科目 = "★要確認"` とする

### 全角→半角正規化

- `unicodedata.normalize("NFKC", text)` を使用

## F-04 結果出力 詳細仕様

### 照合済み出力カラム（matched.csv）

| カラム名 | 内容 |
|----------|------|
| 取引日 | 仕訳帳の取引日 |
| 利用内容 | りそなの利用内容 |
| 金額 | 金額（円） |
| 勘定科目 | ルール推定結果 |
| 補助科目 | ルール推定結果 |
| 税区分 | ルール推定結果 |
| 仕訳帳_摘要 | 仕訳帳の摘要 |

### 未照合出力カラム

- りそなのみ：りそなの全カラム＋`勘定科目`推定結果
- 仕訳帳のみ：仕訳帳の全カラム

## CLIインターフェース

```bash
# F-01: VISAデビ抽出
python -m accounting.extract_visa_debit [入力CSV] [出力CSV]

# F-02〜04: 照合＋勘定科目推定
python -m accounting.reconcile \
  --journal data/仕訳帳.csv \
  --debit   data/risona_debit.csv \
  --rules   accounting/rules/merchant_rules.yml \
  --output  data/output/
```

## 異常系・バリデーション

| 条件 | 挙動 |
|------|------|
| 入力ファイルが存在しない | `FileNotFoundError` を raise し、メッセージを表示して終了 |
| 必須カラムが存在しない | `ValueError` を raise し、欠損カラム名を表示して終了 |
| ルールファイルが存在しない | 警告を表示し、全行を `★要確認` として処理を継続 |
| 金額が数値でない行 | その行をスキップし、警告ログに記録 |
