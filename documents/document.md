# 法人決算支援ツール 仕様書

## 概要

月次の仕訳作業を自動化するCLIツール。
りそなデビットカード明細（CSV）と freee 仕訳帳（CSV）を照合し、
勘定科目ルールに基づいて仕訳済み結果を出力する。

### 全体方針：複数視点で仕訳帳を消化する

仕訳帳の全行を、複数の照合タスクで段階的に消化していく。
各照合タスクでマッチした取引は当該タスクの出力CSVに振り分けられ、
最終的に残った未照合行が `仕訳帳のみ.csv` に集約される。
理想形は `仕訳帳のみ.csv = 0件`。

```
仕訳帳全件
  ├─ カード照合（本ツール F-02〜04）→ カード_照合済み で消化
  ├─ 振込照合（将来）　              → 振込_照合済み で消化
  ├─ 給与照合（将来）　              → 給与_照合済み で消化
  └─ 仕訳帳のみ                       ← 0 を目指す
```

---

## 機能一覧

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| F-01 | VISAデビ抽出 | 仕訳帳から「★要確認 × VISAデビ始まり」の行を抽出 |
| F-02 | カード照合 | りそな明細 × 仕訳帳を金額＋日付で照合 |
| F-03 | 勘定科目推定 | ルールファイルの正規表現で利用内容から勘定科目を推定 |
| F-04 | 結果出力 | 照合結果・未入力・仕訳帳のみをCSVに分けて出力 |

## F-02 カード照合処理 詳細仕様

### 入力ファイル

| ファイル | エンコード | 主要カラム |
|----------|------------|------------|
| りそなデビット明細CSV | CP932 | 利用日, 利用内容, 金額, 承認番号, ステータス |
| freee 仕訳帳CSV | CP932 | 取引日, 借方勘定科目, 借方補助科目, 借方金額(円), 貸方勘定科目, 貸方補助科目, 摘要 |

### 仕訳帳の対象行

りそな普通預金口座を経由するVISAデビ候補を抽出する。MFクラウド連携で
摘要が取引先名に書き換えられたエントリも漏れなく拾うため、口座ベースで判定する。

- `借方補助科目` または `貸方補助科目` が `"りそな銀行大手支店普通0073514"`
- かつ反対側の `勘定科目` が `"普通預金"` または `"現金"` ではない（口座間振替・現金引出を除外）

口座番号は `accounting/reconcile.py` の `RISONA_ACCOUNT` 定数で定義。

### 照合キー

- **金額**：りそな `金額` の絶対値 == 仕訳帳 `借方金額(円)` の絶対値
  - りそな側は返金がマイナス値、仕訳帳側は借方/貸方で方向を表現するため絶対値で比較
- **日付**：りそな `利用日` == 仕訳帳 `取引日`（フォーマット正規化後に比較）

### 日付フォーマット正規化

| 元フォーマット | 正規化後 |
|----------------|----------|
| `2025年04月07日`（りそな） | `2025-04-07` |
| `2025/04/07`（仕訳帳） | `2025-04-07` |

### りそな明細のステータスフィルタ

| ステータス | 扱い |
|---|---|
| 確定 | 照合対象 |
| 未確定（承認分）等 | `カード_未確定.csv` に退避（仕訳化前なので照合対象外） |
| 決済不可 | `カード_対象外.csv` に退避（取引不成立のため対象外） |

### 出力ファイル一覧

| 出力ファイル | 内容 |
|------|------|
| `カード_照合済み.csv` | 金額＋日付で一致したりそなと仕訳帳の対 |
| `カード_照合済み_コンパクト.csv` | 照合済みを入力支援用カラムに絞り込み |
| `カード_重複要確認.csv` | 同日・同金額が複数件で1対1にならず要目視判定 |
| `カード_未入力.csv` | りそな（確定）にあって仕訳帳に未入力 |
| `カード_未確定.csv` | りそな明細でステータスが「確定」以外 |
| `カード_対象外.csv` | りそな明細でステータスが「決済不可」 |
| `仕訳帳のみ.csv` | 全仕訳帳からカード照合で消化された取引Noを除いた残り |

### 仕訳帳のみ.csv の定義

`仕訳帳のみ.csv = 全仕訳帳 − (カード_照合済み の取引No + カード_重複要確認 の取引No)`

カード照合で消化されなかった全仕訳帳行が集約される。本ツールのスコープ外
（社会保険料引落、振込手数料、法人税納付など）も含まれ、将来の振込照合・
給与照合などのツールで段階的に消化していく対象となる。

### 最終評価

カード照合の完了判定は `カード_重複要確認` と `カード_未入力` の合計で行う。
両方が0件なら `〇`。`仕訳帳のみ.csv` の件数は別途表示するが、評価対象には
含めない（他の照合ツールで消化すべき行のため）。

```
カード照合: 〇 すべて照合済み
仕訳帳のみ: 1462件（カード照合外・他ツールでの照合待ち）
```

## F-03 勘定科目推定 詳細仕様

### ルールファイル

- パス：`accounting/rules/merchant_rules.yml`
- フォーマット：YAML

```yaml
rules:
  - match: "ZOOM"          # 正規表現（大文字小文字・全角半角を正規化後にマッチ）
    勘定科目: 消耗品費
    補助科目: Zoom
    税区分: 課税仕入
```

### マッチング手順

1. りそな `利用内容` を **全角→半角** に正規化
2. ルールを上から順に評価し、**最初にマッチしたルールを採用**
3. どのルールにもマッチしない場合は `勘定科目 = "★要確認"` とする

### 全角→半角正規化

- `unicodedata.normalize("NFKC", text)` を使用

## F-04 結果出力 詳細仕様

### カード_照合済み.csv

りそな全カラム＋仕訳帳全カラム（結合時に重複しないため `_journal` サフィックスは付かない）
＋勘定科目推定結果（`推定入力名` `推定勘定科目` `推定補助科目` `推定税区分`）
＋入力補助列（`利用内容（入力）` `借方勘定科目（入力）` `補助科目（入力）`）
＋ `要目視確認` フラグ（同日同額の複数件マッチ時に「要確認」）

### カード_照合済み_コンパクト.csv

freeeへの入力作業時に確認しやすいカラムのみ抽出した版。
カラム順は `accounting/reconcile.py` の `COMPACT_COLS` で定義。

### カード_未入力.csv

りそな全カラム＋勘定科目推定結果。仕訳帳に未転記なので、
推定結果を参考に freee へ入力していく。

### カード_未確定.csv / カード_対象外.csv

りそな全カラム（フィルタのみ、加工なし）。

### 仕訳帳のみ.csv

仕訳帳の全カラム（フィルタのみ、加工なし）。

## CLIインターフェース

すべてのオプションは省略可能。省略時は下記デフォルトが適用される。

```bash
# F-01: VISAデビ抽出
python -m accounting.extract_visa_debit [入力CSV] [出力CSV]

# F-02〜04: 照合＋勘定科目推定
python -m accounting.reconcile \
  [--journal data/accounting/仕訳帳.csv] \
  [--debit   data/accounting/risona_debit.csv] \
  [--rules   accounting/rules/merchant_rules.yml] \
  [--output  data/accounting/output/]
```

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `--journal` | `data/accounting/仕訳帳.csv` | freee仕訳帳CSV（CP932） |
| `--debit` | `data/accounting/risona_debit.csv` | りそなデビット明細CSV（CP932） |
| `--rules` | `accounting/rules/merchant_rules.yml` | 勘定科目ルールYAML |
| `--output` | `data/accounting/output/` | 結果CSVの出力先ディレクトリ |

通常運用では引数なしで実行可能。

```bash
python -m accounting.reconcile
```

## ディレクトリ構成

コードと設定はツール名のフォルダ（例: `accounting/`）に閉じ込め、実データは
プロジェクトルートの `data/<ツール名>/` 配下に置く。新しいツールを追加する際も
同様の規則に従う（`<ツール名>/` にコード、`data/<ツール名>/` にデータ）。

```
hana-agent/
├── accounting/                       # コード＋設定
│   ├── __init__.py
│   ├── extract_visa_debit.py         # F-01
│   ├── reconcile.py                  # F-02〜04 CLIエントリーポイント
│   ├── matcher.py                    # F-02 照合ロジック
│   ├── classifier.py                 # F-03 勘定科目推定ロジック
│   └── rules/
│       └── merchant_rules.yml
├── data/                             # 実データ（gitignore）
│   └── accounting/
│       ├── 仕訳帳.csv
│       ├── risona_debit.csv
│       └── output/                   # 実行時に自動生成
├── tests/
│   ├── unit/
│   │   ├── test_matcher.py
│   │   └── test_classifier.py
│   ├── integration/
│   │   └── test_reconcile.py
│   └── conftest.py
├── documents/
│   └── document.md
├── pyproject.toml
├── .env
└── .env.example
```

## 異常系・バリデーション

| 条件 | 挙動 |
|------|------|
| 入力ファイルが存在しない | `FileNotFoundError` を raise し、メッセージを表示して終了 |
| 必須カラムが存在しない | `ValueError` を raise し、欠損カラム名を表示して終了 |
| ルールファイルが存在しない | 警告を表示し、全行を `★要確認` として処理を継続 |
| 金額が数値でない行 | その行をスキップし、警告ログに記録 |
